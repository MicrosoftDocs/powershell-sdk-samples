Introduction

Windows PowerShell 3.0 SDK Sample Pack contains code samples that show how to build applications based on Windows PowerShell 3.0.

Windows PowerShell is a Microsoft .NET Framework-connected environment designed for administrative automation. Windows PowerShell provides a new approach to building commands, composing solutions, and creating graphical user interface-based management tools.

Overview

This sample pack contains code samples that demonstrate how to use the Windows PowerShell 3.0 API to build a rich set of applications. In this package, you will find code samples that show how to host the Windows PowerShell Workflow runtime, generate workflow activities from Windows PowerShell cmdlets and modules, disconnect and reconnect a Windows PowerShell session, support paging operations, and a number of other tasks that are possible with Windows PowerShell 3.0. This sample pack also includes samples from the Windows PowerShell 2.0 SDK.

The following code samples are new for Windows PowerShell 3.0:

  • Activity Controller Extensibility
  • Activity Generator
  • Disconnected Sessions
  • Job Source Adapter
  • Script Line Profiler
  • SQL Store Extensibility
  • Supports Paging
  • Workflow Hosting

The following code samples from previous releases of Windows PowerShell are also included:

  • Event Registration
  • Event Notification
  • Hosts
  • PowerShell
  • Cmdlets
  • Providers
  • Runspaces
  • Remote Runspaces
  • Serialization

Requirements

Using the samples requires that you have the following installed:

  • Windows PowerShell 3.0
  • Visual Studio 2010 or later
  • Windows 8 SDK, available at http://msdn.microsoft.com/en-us/windows/hardware/hh852363.aspx

Note: After installing the Windows 8 SDK, the Windows PowerShell reference assemblies will be installed in the %ProgramFiles(x86)%\Reference Assemblies\Microsoft\WindowsPowerShell\3.0 folder.
